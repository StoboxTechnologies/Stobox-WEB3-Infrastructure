Architecture

Diamond Standard for maximum modularity.

Token Offering Module is built on the Diamond Pattern (EIP-2535), enabling seamless functionality upgrades without losing state or history. Each facet is responsible for a specific aspect of offering management.

DiamondCutFacet

Handles system upgrades and facet management. Add, replace, or remove functionality through an admin-controlled upgrade mechanism with state preservation.

DiamondLoupeFacet

Introspection and querying capabilities: view available facets and functions, system transparency, and debugging tools for development and integration.

RegistryRoleFacet

Access control and permission management: role assignment and revocation, multi-role support (admin, tech service, offering manager) with clear hierarchies.

OfferingGovernanceFacet

STO lifecycle management: creation, activation, suspension, cancellation, and termination of offerings. State transition controls and governance operations.

RegistryStorageFacet

Data storage and retrieval operations: offering metadata management, purchase record storage, and historical data access with comprehensive query capabilities.

RegistryRuleEngineFacet

Rule enforcement and compliance validation: link/unlink rules to offerings, validate investor eligibility, and enforce investment limits with custom logic.
